# Roadmap

> The **strategy** map — milestones only. Every milestone is **a thing you can play**, not a
> system that exists. Vertical slice before breadth (GDD §12).
>
> The active milestone is decomposed into ~1-hour daily tasks in its own backlog under
> [`docs/backlog/`](backlog/) (just-in-time — only one backlog is alive at a time; the rest
> stay coarse here until reached). This file changes rarely; the active backlog is the
> living doc.

## How milestones pass

**Checkpoint gate 🚧** — a milestone isn't *passed* until: tasks done → build played through
the [Fun Checklist](FUN_CHECKLIST.md) lenses → a **fun verdict** written in
[PLAYTESTS.md](PLAYTESTS.md). "Not fun" → iterate or cut, don't advance. Gates apply only at
**playable boundaries** (M0 has nothing to drive → no gate).

**Standing rules** (carried into every backlog): mix code + asset tasks to keep momentum ·
keep quick wins for low-energy days · performance-first from the start · throwaway work lives
in `prototypes/` (excluded from release) · lean on CC0 + MagicaVoxel, log in CREDITS.
